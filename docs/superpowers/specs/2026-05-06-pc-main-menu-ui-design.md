# Goals PC 主菜单 UI 改造设计

## 1. 背景

`goals` 项目当前桌面端主菜单位于左侧 `Sidebar`，采用传统后台式纵向列表：左图标、右文字，并带有“展开 / 收起”两态。移动端 `MobileNavBar` 已经采用“上 icon / 下文字”的入口式表达。

本次目标是仅优化 `goals` 的 PC 端主菜单 UI，参考可灵 AI 的轻面板风格，但不做整条侧栏的全面重设计。用户明确要求：

- 采用类似移动端的“上 icon / 下文字”展示
- 采用方案 A：主菜单卡片化
- 去掉当前侧栏“展开 / 收起”两态
- 仅改主菜单区，不重做品牌区和底部退出区

## 2. 当前状态分析

### 2.1 相关文件

- `src/components/Sidebar.tsx`
  - 包含桌面端侧栏主体
  - 当前维护 `isCollapsed`、tooltip、折叠按钮、导航项渲染
- `src/components/MobileNavBar.tsx`
  - 已有移动端“上 icon / 下文字”的结构，可作为信息层级参考
- `src/app/(authenticated)/layout.tsx`
  - 以 `<Sidebar dict={dict} />` 方式接入侧栏，接口简单稳定

### 2.2 当前问题

- 侧栏存在双态，视觉与交互成本较高，不符合本次“更像 AI 产品面板”的目标
- 主菜单是传统列表式，入口感较弱
- 收起态依赖 tooltip 与 portal，逻辑复杂度高于实际价值
- PC 菜单与移动端导航的视觉语言不一致

## 3. 设计目标

- 将桌面端主菜单改造成更像“功能入口面板”的导航区
- 保持风格克制、轻量、现代，接近可灵 AI 的体验气质
- 移除展开/收起两态，保留单一稳定版式
- 不新增依赖，不改 `Sidebar` 对外接口，不影响移动端导航

## 4. 非目标

- 不重做顶部品牌区结构
- 不重做底部退出区流程
- 不调整移动端 `MobileNavBar` 与 `MobileSidebar`
- 不引入新的导航信息架构
- 不新增动画系统、设计 token 或依赖库

## 5. 方案决策

### 5.1 采用方案

采用“固定宽度单态侧栏 + 2 列卡片式主菜单”的方案。

### 5.2 决策说明

#### 背景

需要让 PC 主菜单更像 AI 产品入口面板，同时与移动端视觉语言对齐。

#### 约束

- 只改主菜单区
- 当前布局通过 `Sidebar` 接入，接口不宜变化
- 不能引入高风险重构

#### 方案

- 删除折叠态及其相关逻辑
- 展开态改为唯一态
- 将主菜单改成 2 列卡片 grid，每张卡片 `icon` 在上、文字在下

#### 取舍

- 优点：更现代、统一、稳定，入口识别效率更高
- 缺点：侧栏将永久占用固定宽度，不再通过折叠换取内容区空间

#### 风险

- 固定宽度需要在“信息舒适度”和“内容区占用”之间平衡
- 卡片式导航如果样式过重，容易喧宾夺主

#### 建议

侧栏宽度控制在约 `220px~240px` 区间，采用轻边框、浅底、弱阴影的卡片体系，避免大面积渐变和重发光效果。

## 6. 信息架构与布局

### 6.1 侧栏整体

- 保留左侧固定侧栏结构
- 桌面端改为单态：不再出现折叠按钮
- 侧栏宽度固定，不再根据状态在 `w-16 / w-64` 之间切换

### 6.2 顶部品牌区

- 保持现有品牌区结构
- 允许做轻微 spacing / 对齐优化，以便与新主菜单风格衔接
- 不重新设计 logo 结构和品牌文案布局

### 6.3 主菜单区

- 从单列 list 改为 `2 列 grid`
- 每个导航入口为统一高度的小卡片
- 卡片内容垂直排列：
  - 上方：icon
  - 下方：label
- 文字居中对齐
- 每个卡片的点击区域完整覆盖整个卡片

### 6.4 底部退出区

- 保留当前底部退出能力
- 不改变退出确认逻辑
- 仅允许做轻量的 spacing / 对齐调整

## 7. 视觉设计

### 7.1 风格关键词

- 轻
- 净
- 软
- 聚焦

### 7.2 卡片默认态

- 大圆角
- 浅背景
- 弱边框
- 不使用重色块
- hover 时轻微提亮、轻阴影

### 7.3 激活态

- 使用主色低饱和浅底
- icon 与文字切换到主色系
- 增加轻边框或轻阴影强化选中态
- 不使用重渐变、不使用强发光

### 7.4 文本与图标层级

- icon 稍大于当前侧栏列表模式
- 文字尺寸略小于传统菜单列表文字
- 保证入口可扫读，不追求长文案承载

## 8. 交互设计

### 8.1 删除的交互

- 删除展开/收起按钮
- 删除 `isCollapsed` 状态
- 删除 tooltip 浮层及其 hover 显示逻辑

### 8.2 保留的交互

- 点击导航项跳转
- 当前页面高亮
- hover 反馈
- 底部退出确认弹窗

### 8.3 响应式边界

- 仅影响桌面端侧栏
- 移动端 `MobileNavBar` 保持不变
- 移动端侧边抽屉保持不变

## 9. 实现策略

### 9.1 改动文件

主要改动：

- `src/components/Sidebar.tsx`

不改动：

- `src/components/MobileNavBar.tsx`
- `src/components/MobileSidebar.tsx`
- `src/app/(authenticated)/layout.tsx`

### 9.2 具体改动项

1. 删除与折叠态相关的状态、按钮、tooltip 逻辑
2. 固定侧栏宽度
3. 将主菜单容器从单列 list 改为 2 列 grid
4. 将导航项从“横向排列”改为“竖向卡片”
5. 保持路由高亮判定逻辑不变
6. 仅对顶部品牌区和底部退出区做必要的视觉衔接调整

### 9.3 接口稳定性

- `Sidebar` 组件 props 不变
- 布局接入方式不变
- 路由和字典字段结构不变

## 10. 验收标准

- PC 端侧栏只保留单态，不再出现展开/收起按钮
- 主菜单改成 2 列卡片式入口
- 每个菜单项为“上 icon / 下文字”
- 激活态清晰但不过重
- 移动端导航无回归
- 不新增依赖
- `Sidebar` 对外接口不变

## 11. 验证清单

- 桌面端打开 `/dashboard`、`/today`、`/goals`、`/potential`、`/profile`
  - 当前路由高亮正确
  - 卡片 hover 与 active 态正确
  - 文案不溢出、不裁切异常
- 页面切换后侧栏布局稳定，不闪烁
- 移动端底部导航与移动侧边栏未受影响
- 退出按钮与确认弹窗功能正常

## 12. 实施备注

- 本次优先保证主菜单风格升级和交互简化
- 若第一版视觉仍需更接近可灵 AI，可在后续迭代中微调：
  - 卡片圆角
  - 阴影层次
  - 主色激活态对比度
  - 品牌区与主菜单间距关系
