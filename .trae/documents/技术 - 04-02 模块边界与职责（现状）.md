# 模块边界与职责（现状）

## 1. 约定：UI / Server Actions / Route Handlers / Lib
- UI：React Server/Client Components（页面与组件）
- Server Actions：以 `actions.ts` 形式提供写操作（被 UI 直接调用）
- Route Handlers：`src/app/api/**/route.ts`（HTTP API）
- Lib：纯函数/工具/封装（可被 UI/Actions/API 复用）

## 3. 边界校验（可作为重构护栏）
- UI 不直接持有 service role key；高权限只允许出现在服务端 actions/handlers。
- 写操作优先收敛在 Server Actions（页面内直调）或 Route Handlers（对外 HTTP），避免散落在组件中。
- Lib 只做纯逻辑/封装，不读写浏览器状态；需要 cookie/session 的逻辑放在 server/action/api 层。
