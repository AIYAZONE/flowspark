# 指标体系与仪表板（MVP → Phase 2）

## 1) 业务目标与北极星（North Star）
- 北极星：心流时长（Flow Time）或“连续进入专注模式的天数”
  - 说明：如果 Phase 2 尚未上线专注模式，可先用“连续 3 天设置核心行动”作为代理指标

## 2) 指标树（从产品到商业）
### 2.1 激活（Activation）
- A1：注册后 10 分钟内完成「目标 + 行动 + 今日核心行动 + 评分」中的 3 项
- A2：首次会话完成 1 次“核心行动设置”

### 2.2 留存（Retention）
- R1：D1 / D7 / D30（按注册 cohort）
- R2：连续 3 天设置核心行动占比
- R3：7 天内至少 3 次评分/复盘占比

### 2.3 价值（Value / Engagement）
- V1：每周活跃目标数（Active Goals）
- V2：每周完成行动数（Actions Completed）
- V3：评分趋势参与率（7 天内评分 ≥ 3 次）

### 2.4 变现（Monetization，Phase 2+）
- M1：付费转化率（Plus/Pro）
- M2：AI 功能复用率（不是“试一次”）
- M3：付费留存（订阅续费/流失）

## 3) 关键事件（Event Taxonomy）
### 3.1 账号与激活事件
- signup_success
- login_success
- goal_created
- action_created
- core_action_set
- daily_score_submitted

### 3.2 留存与价值事件
- action_completed
- goal_archived
- dashboard_viewed

### 3.3 Phase 2 事件（规划）
- ai_breakdown_requested
- ai_breakdown_accepted
- reward_opened
- level_up
- zen_mode_started
- zen_mode_ended

## 4) 数据采集建议（不新增依赖的前提）
- 性能与 Web Vitals：使用现有的 Vercel Analytics（已集成）
- 产品事件：优先在服务端动作（Server Actions）或 Supabase 函数侧写入事件表（避免纯前端埋点丢失）
  - 事件表最小字段建议：event_name、user_id、occurred_at、properties(jsonb)、request_id

## 5) 仪表板布局（一个页面就够用）
### 5.1 产品健康（每日/每周）
- 激活：A1、A2
- 留存：R1、R2
- 价值：V1、V2、V3

### 5.2 性能与稳定性
- LCP/TTFB（登录、仪表盘、目标列表）
- 认证成功率、关键动作失败率（submitScore、toggleAction 等）
