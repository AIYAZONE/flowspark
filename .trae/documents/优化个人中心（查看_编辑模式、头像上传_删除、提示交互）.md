## 目标与补充要求
- 查看/编辑双模式
- 头像：选择后本地预览；真正上传在“更新资料”提交；可删除
- 成功/失败提示：Toast，3s隐藏，位置统一右上
- 新注册用户默认头像：在注册后或首次登录时为用户落地一个默认头像 URL

## 默认头像策略
- 默认头像来源：项目静态资源或公开 CDN（如：/images/default-avatar.png 或 https://cdn.example.com/default-avatar.png）
- 落地时机（二选一，兼容实现）：
  1) 注册成功后（signup Server Action 内）写入 user_profiles.avatar_url 与 auth.user metadata.avatar_url
  2) 首次访问 /profile 时，如果缺少 avatar_url，则由 Server Action 写入默认 URL
- 安全与一致：写入 user_profiles，并同步到 auth 用户 metadata 作为兜底

## 页面与组件
- ProfileView（客户端）：展示资料与“编辑”按钮
- ProfileEditor（客户端）：表单（姓名、时区、头像选择预览、移除开关），提交/取消
- AvatarUploader（客户端）：仅本地预览与选择，不做即时上传，向父组件暴露 selectedFile 与 remove 标记

## 服务端动作
- updateProfile：
  - 处理 name、timezone、locale
  - avatar_control=upload/remove/none；当 upload 并有 avatar_file，则上传到 avatars/{user.id}/{timestamp}.{ext}，写入 user_profiles.avatar_url
  - 当 remove，则解析现有 URL 删除存储文件并清空 avatar_url（删除失败仅清空资料）
- ensureDefaultAvatar（可复用）：首次进入 /profile 或注册后调用，若缺失则写默认 URL

## 提示交互
- Toast（右上）：更新成功/失败、头像上传/删除成功/失败；3s 自动隐藏
- 编辑提交成功后切回查看模式并刷新数据

## i18n 文案
- profile: edit/cancel/save，avatar: change/remove/defaultApplied，toast: updateSuccess/updateFailed/avatarRemoved/avatarUploadFailed
- zh/en 两套
