## 背景
你现在的“今日行动”把所有覆盖今天的 action 平铺出来（含跨天进行中 + 延期未完成），当目标/行动多时会形成很长的列表，导致认知负担高、产生“任务好多”与挫败感。

## 约束
- 现有数据模型：`actions` 只有 `type(core/maintenance/learning/review/rest)`、`priority(high/medium/low)`、`completed`、日期范围等字段，没有 pin/focus 字段（因此优先用“纯 UI 策略”实现聚焦）。
- 现有页面：[/src/app/(authenticated)/today/page.tsx](file:///Users/brucewang/Documents/AIYA/goals/src/app/(authenticated)/today/page.tsx) 直接使用 [ActionListFilter](file:///Users/brucewang/Documents/AIYA/goals/src/components/ActionListFilter.tsx) 平铺渲染。
- 已有可复用折叠能力：项目内已有 [Collapsible](file:///Users/brucewang/Documents/AIYA/goals/src/components/ui/collapsible.tsx) 和业务用法参考（[GoalListFilter.tsx:L276-L350](file:///Users/brucewang/Documents/AIYA/goals/src/components/GoalListFilter.tsx#L276-L350)）。
- 必须保持现有接口行为不被破坏：`ActionListFilter` 在目标详情页等也被使用。

## 方案
### 方案 A（推荐）：不改数据库，做“聚焦视图 + 分组折叠 + 渐进披露”
1. 在“今日行动”页新增一个客户端容器组件（例如 `TodayActionList`）：
   - **默认展示聚焦区**：
     - “必须完成”：优先显示 `priority=high` + `type=core` 的未完成（不足则补充其他未完成），最多 N 条（建议 6~8）。
     - “延期未完成”：单独折叠分组（默认展开，但只显示前 N 条 + Show more）。
   - **其他行动**：按目标分组（goal title），每组用 Collapsible 折叠，默认收起，只显示数量。
   - **已完成**：整体折叠（默认收起）。
2. 提供一组“快捷筛选 chips”（不替代现有高级筛选）：
   - 仅看必须（core+high）、仅看高优先级、仅看核心、仅看延期、显示全部。
3. 顶部增加“摘要条”降低焦虑：
   - 展示未完成数量、必须数量、延期数量、已完成数量；并在“过载”时提示“建议今日只保留 6~8 个行动”。
4. 复用现有 ActionItem（不牺牲编辑/完成/删除能力），保证行为一致。

### 方案 B（次优但更强）：增加 `actions.is_focus` 落库
- 新增字段 + RLS + UI pin 按钮；今日默认先展示 is_focus=true。
- 成本更高（迁移/RLS/更多代码改动），但更符合长期使用。

## 取舍
- 方案 A：改动小、上线快、无需迁移，立即缓解“列表太长”的压力；缺点是“聚焦”是 UI 策略而非用户显式 pin。
- 方案 B：用户可明确指定焦点任务，长期更精确；缺点是需要数据库迁移与 RLS 调整，风险与回归面更大。

## 风险
- “折叠/限量”可能让用户担心任务被隐藏：需要明显的数量提示与“显示全部”入口。
- 分组/聚焦逻辑若与用户心智不一致会增加学习成本：需要默认策略简单、并允许一键回到“显示全部”。

## 建议
先落地方案 A（低风险、立竿见影），用真实使用反馈验证后再决定是否上方案 B。
