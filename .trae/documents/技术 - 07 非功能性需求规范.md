# 非功能性需求规范（NFR）

## 性能
- 页面性能预算：LCP ≤ 2.5s（移动 4G），CLS ≤ 0.1，TBT ≤ 200ms
- 关键路径：登录、仪表盘、今日行动、目标列表与详情
- 资源策略：按需加载、代码分割、图标与图片优化（SVG/压缩）

## 可靠性
- 可用性目标：月度可用性 ≥ 99.9%
- 重试与降级：服务异常时提供只读视图和重试提示
- 数据一致性：服务端动作确保事务性与失败回滚

## 安全
- RLS 全面启用，owner_id = auth.uid() 的读写限制
- 密钥管理：仅使用 Anon Key；禁止在客户端暴露 Service Role Key
- 速率限制：重置邮件与登录尝试节流（同邮箱/同 IP 时间窗限制）

## 可维护性
- 代码质量：类型严格、Lint 与格式化、单测与 E2E 测试覆盖
- 监控与日志：基础日志（不含敏感信息）、关键指标追踪与告警

## 可用性
- 国际化：中文/英文切换；错误与成功提示一致性
- 无障碍：语义化标签、aria 属性、键盘导航支持
