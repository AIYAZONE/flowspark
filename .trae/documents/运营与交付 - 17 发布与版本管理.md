# 发布与版本管理

## 版本规则
- 语义化版本：MAJOR.MINOR.PATCH；破坏性变更需记录迁移指南

## 流程
- 分支策略、CI 构建、预览验证、生产发布、回滚与灰度
  - 发布前检查清单：
    - 本地执行 `npm run build`，确保 Next.js 编译与 TypeScript 校验通过
    - i18n 键一致性检查：新增或使用的键必须存在于所有语言字典中（如 `common.locale`、`common.noData`、`today.alreadyScored` 等）
    - 行动类型一致性：UI、Server Actions 与数据字典中的 `action.type` 枚举保持一致
    - 时区逻辑复核：`getUserTimezone/getTodayInTZ/toLocaleDateStringTZ` 在涉及“今日”逻辑处使用
  - Vercel 构建环境：
    - Next.js 16.1（Turbopack），Node 版本由 Vercel 托管
    - 构建日志常见错误：类型不匹配（字典键缺失）、接口入参不一致（如 `submitScore` 使用 `{date, score}`）

## 变更记录
- 与 CHANGELOG 联动：摘要、影响范围、回滚指引
  - 示例（2025-12-30）：
    - 影响范围：today、dashboard、i18n
    - 回滚指引：移除新增 i18n 键并恢复旧枚举；改回本地时间逻辑（不推荐）
