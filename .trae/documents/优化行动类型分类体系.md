# 行动类型优化与扩展计划

您提到目前的“核心 (Core)、成长 (Growth)、日常 (Routine)”分类不能满足多样化的行动需求。为了更好地覆盖各种场景，我建议采用一种基于**“行动性质”与“能量消耗”**的混合分类法，既能区分任务的重要性，又能反映任务的属性。

## 1. 拟定新类型 (Proposed New Types)

我们将扩展为 5 种类型，覆盖从高强度攻坚到低强度恢复的全谱系：

1.  **攻坚 (Deep Work)**
    *   *定义*：需要高度专注、消耗大量脑力、对目标有决定性推进作用的任务。
    *   *替代*：Core (核心)
    *   *场景*：撰写核心代码、完成重要报告、深入学习新技能。

2.  **推进 (Progress)**
    *   *定义*：常规的进度推进，按部就班完成的任务。
    *   *替代*：Growth (成长) 的部分含义
    *   *场景*：日常开发、跟进项目进度、阅读文档。

3.  **维持 (Maintenance)**
    *   *定义*：必须做但无需太多创造力的事务性工作，维持系统或生活运转。
    *   *替代*：Routine (日常)
    *   *场景*：回复邮件、整理文件、家庭琐事、系统备份。

4.  **学习 (Learning)**
    *   *定义*：纯粹的输入型任务，为了未来储备能力。
    *   *新增*
    *   *场景*：看书、听课、练习外语、技术调研。

5.  **健康 (Health/Rest)**
    *   *定义*：恢复精力、锻炼身体或关注身心健康的任务。
    *   *新增*
    *   *场景*：健身、冥想、散步、早睡。

## 2. 实施步骤 (Implementation Steps)

1.  **更新多语言文件 (`src/i18n/*.json`)**:
    *   移除旧的 `core`, `growth`, `routine`。
    *   添加新的 `deep_work`, `progress`, `maintenance`, `learning`, `health` 及其对应的中英文翻译。

2.  **更新组件**:
    *   修改 `AddActionForm.tsx` (添加行动表单)
    *   修改 `AddActionDialog.tsx` (详情页弹窗)
    *   修改 `ActionItem.tsx` (行动项展示与编辑)
    *   *操作*：将下拉菜单的选项更新为上述 5 个新类型。

3.  **数据兼容性 (Data Compatibility)**:
    *   *注意*：数据库中已存在的旧类型数据 (`core`, `growth`, `routine`) 仍然会保留。为了不破坏显示，我们将在代码中保留对旧类型的回退支持（Fallback），或者在 UI 上将旧类型映射到最接近的新类型（例如 core -> deep_work）。
    *   *策略*：新创建的行动使用新类型。旧行动在编辑时，用户可以选择切换到新类型。

## 3. 预期效果

*   **更精准**：您可以更准确地标记任务是“攻坚”还是“维持”，从而更好地安排精力。
*   **更全面**：增加了“学习”和“健康”，体现了个人成长不仅仅是工作，还包括自我提升和身心平衡。
