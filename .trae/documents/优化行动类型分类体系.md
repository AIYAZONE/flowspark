# 行动类型优化计划 (基于目标分类)

您指出了一个非常关键的点：**行动 (Action) 应该与目标 (Goal) 的分类相呼应**。既然行动是为了实现目标，那么行动的类型也应该反映它在实现该类目标过程中的角色。

## 1. 核心理念：通用维度 + 领域特定

我们现有的目标分类包括：`健康`, `事业`, `学习`, `财务`, `生活`, `社交`。
为了让行动类型既能对应目标，又具备通用性（毕竟不同目标的行动也有共性），我建议采用\*\*“四象限行动模型”\*\*，这套模型可以套用到任何目标分类上：

### 新的行动类型 (New Action Types)

1. **核心攻坚 (Deep Work / Core)**

   * *含义*：直接产出结果、难度最大、最重要的任务。

   * *对应目标示例*：

     * 事业 -> 完成项目代码

     * 学习 -> 研读专业书籍

     * 健康 -> 高强度训练

2. **日常维持 (Maintenance)**

   * *含义*：为了保持现状或为攻坚做准备的重复性、事务性任务。

   * *对应目标示例*：

     * 事业 -> 回复邮件、周报

     * 财务 -> 记账

     * 健康 -> 准备健康餐

3. **学习探索 (Learning / Explore)**

   * *含义*：获取新知、调研、寻找新方法的任务。

   * *对应目标示例*：

     * 事业 -> 调研新技术

     * 社交 -> 参加新圈子聚会

     * 生活 -> 寻找旅游攻略

4. **复盘思考 (Review / Reflect)**

   * *含义*：总结经验、调整方向、思考策略的任务。

   * *对应目标示例*：

     * 所有目标 -> 周复盘、月度规划

5. **休息恢复 (Rest / Health)**

   * *含义*：主动的休息、放松，为了更好地出发。

   * *对应目标示例*：

     * 所有目标 -> 冥想、散步、早睡

## 2. 实施方案

### 步骤 1: 更新多语言文件 (`i18n/*.json`)

* 保留 `core` (作为核心攻坚)，`maintenance` (日常维持)，`learning` (学习探索)，`health` (休息恢复)。

* 新增 `review` (复盘思考)。

* 移除旧的 `growth`, `routine` (确认它们没有对应的数据)。

### 步骤 2: 更新 UI 组件

修改以下组件的下拉菜单，使用新的类型列表：

* `src/components/AddActionForm.tsx`

* `src/components/AddActionDialog.tsx`

* `src/components/ActionItem.tsx`

### 步骤 3: 智能默认值 

虽然行动类型是通用的，但我们可以根据选择的目标，在创建行动时提供**智能提示**或**默认值**。

* 例如，如果选择了“学习”类的目标，行动类型默认倾向于“学习探索”或“核心攻坚”。

## 3. 预期变化

您的下拉菜单将变为：

1. **核心攻坚** (Core/Deep Work)
2. **日常维持** (Maintenance)
3. **学习探索** (Learning)
4. **复盘思考** (Review)
5. **休息恢复** (Rest)

这样无论您的目标是“减肥”（健康类）还是“升职”（事业类），您都可以清晰地定义当下的行动是属于“核心攻坚”（如去健身房/写PPT）还是仅仅是“日常维持”（如买菜/回消息）。
