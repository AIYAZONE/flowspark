# 技术 - 21 AI分析二期设计

## 1. 目标

在现有 `AI Analytics` 总览页基础上，进入第二阶段：

- 支持按 `scene` 下钻分析
- 支持查看单条 `recommendation` 的明细与结果回放
- 支持更可解释地回答：
  - 哪个 scene 更有效
  - 哪个 strategy / prompt / model 组合更有效
  - 某条 recommendation 为什么出现、最终结果如何

本阶段不新增一级导航，不引入外部 BI 平台，不改现有 recommendation 数据结构。

## 2. 设计原则

- **总览与下钻分层**：总览页负责发现问题，下钻页负责解释问题
- **结构化优先**：优先展示 recommendation 的结构化输出和结果，不展示原始长 prompt
- **可回放**：任一 recommendation 都能看到生成信息、采用结果、完成状态
- **最小侵入**：复用现有 `/profile/ai-insights`、`analyticsStore`、`WeeklyInsightCard`

## 3. 信息架构

### 3.1 页面结构

- 总览页：`/profile/ai-insights`
- 场景详情页：`/profile/ai-insights/[scene]`
- recommendation 明细：第一版使用页面内详情抽屉/详情卡，不额外建顶级路由

### 3.2 导航流

1. 用户进入 `/profile/ai-insights`
2. 在 scene 指标表中点击某个场景
3. 进入 `/profile/ai-insights/[scene]`
4. 查看该 scene 下：
   - 总量 / 采纳率 / 完成率 / fallback
   - strategy / prompt / model 对比
   - 最近 recommendation 列表
5. 点击某条 recommendation
6. 打开 recommendation detail，查看输出摘要与 outcome

## 4. 数据设计

### 4.1 沿用现有表

- `ai_recommendations`
- `ai_recommendation_outcomes`

### 4.2 新增读取能力

不新增核心业务表，优先新增读取层：

- `getAISceneDetailMetrics(scene, filters)`
- `getAIRecommendationsByScene(scene, filters)`
- `getRecommendationDetail(recommendationId)`

### 4.3 recommendation detail 输出字段

第一版展示：

- recommendation 基本信息
  - `scene`
  - `strategy_version`
  - `prompt_version`
  - `model`
  - `confidence`
  - `fallback_used`
  - `created_at`
- output 摘要
  - `today_plan`: recommendation kind / variants / reason
  - `rescue`: reason_tag / minimal_variant / if_then
  - `review`: summary_sentence / friction / tomorrow_card
  - `weekly_insight`: summary / momentum / recommendation
- outcome
  - `adopted`
  - `completed`
  - `option_selected`
  - `feedback_label`
  - `completion_minutes`

## 5. 页面设计

### 5.1 总览页增强

在现有 `/profile/ai-insights` 上：

- 保留时间范围筛选与排序
- scene 表增加可点击入口
- recent recommendations 增加“查看详情”动作

### 5.2 scene 详情页

页面结构：

1. 顶部信息区
   - 场景名称
   - 当前时间范围
   - 返回总览

2. 核心指标卡
   - recommendation 数
   - adoption rate
   - completion rate
   - fallback rate

3. strategy / prompt 对比

4. model 对比

5. recent recommendations 列表

6. recommendation detail 面板

### 5.3 recommendation detail 面板

采用右侧抽屉或页内详情卡，展示：

- 生成版本信息
- 输出摘要
- 采用/完成/关闭状态
- 若是 `today_plan`，显示 5/10/20 变体
- 若是 `review`，显示 tomorrow card

## 7. 风险与控制

### 风险 1：详情展示过于技术化

控制：

- 不直接展示原始 JSON
- 先做面向产品的结构化摘要卡

### 风险 2：总览页继续变重

控制：

- 详情不堆在总览页中
- 使用 scene 路由承接

### 风险 3：scene 聚合与总览聚合口径不一致

控制：

- 总览与详情都走同一 `analyticsStore`
- 聚合口径写死在 service 层 helper 中

## 8. 验收标准

- 用户可以从总览页进入任意 scene 详情
- 用户可以查看单条 recommendation 明细
- 可以回答“这条 recommendation 生成了什么、后来如何”
- 可以回答“某个 scene 下哪个 strategy / model 更有效”
- `npm run build` 通过
